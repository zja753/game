# 土豆兄弟风格游戏 —— 开发路线

> 引擎:Excalibur.js + React(已搭好最小场景:玩家、WASD 移动、追踪敌人、墙、相机锁主角、敌人生成定时器)。
> 本路线按"先把土豆兄弟核心循环跑通,再补内容和打磨"的顺序排列,每个里程碑可独立验证。

## 0. 里程碑总览

| #   | 里程碑                               | 关键验证                            |
| --- | ------------------------------------ | ----------------------------------- |
| M0  | 战斗核心:武器 / 投射物 / 受伤 / 死亡 | 玩家能打、能掉血、死了能重开        |
| M1  | 拾取 & 经验 & 升级                   | 怪死后掉经验,捡满升级,可选 1 个升级 |
| M2  | 关卡结构:波次 / 倒计时 / 传送门      | 倒计时结束出传送门,进门进入下一关   |
| M3  | 多样敌人:类型 / 行为 / 属性          | 至少 3 种敌人,各有一种行为模式      |
| M4  | 多样武器                             | 至少 3 种武器,共享投射物系统        |
| M5  | HUD & 商店 & 升级选择 UI             | 暂停、升级三选一、关卡间商店        |
| M6  | 角色 / 初始能力                      | 至少 1 个可选角色,带被动            |
| M7  | 音效 & 视觉打磨                      | 命中反馈、击杀粒子、HUD 动效        |
| M8  | 平衡 & 内容扩展                      | 数值调优、增加关卡数与敌人种类      |

每一阶段结束都必须:运行 `vp dev` / `vp check`,在浏览器里走一遍该阶段交付的所有交互,留一个手测脚本在 `game/plan/` 下。

## M0. 战斗核心

**目标**:玩家有"打"的能力,敌我有生死。
**现状缺口**:没有武器、没有血量、没有碰撞伤害、没有死亡处理。

- 新增 `Health` 组件(组件式,挂在 Actor 上),含 `hp / maxHp / invulnerableTimer`。
- 新增 `Weapon` 抽象:持武器时按攻击键(空格 / 鼠标左键)按 `fireRate` 周期发射投射物。
- 新增 `Projectile` Actor:拥有 owner(避免击中自己)、`damage / speed / lifetime / pierce`。
- 玩家与敌人接触(`onCollisionStart`)扣血,玩家短暂无敌(`invulnerableTimer`)。
- 玩家血量归零:暂停场景、显示 "Game Over" 面板、按钮回到 M2 之前的最小可玩状态(玩家满血、敌人清空、计时归零)。
- 投射物与墙碰撞销毁(避免穿出世界)。

**验证**:屏幕中玩家能朝最近敌人方向开火;被围时血条会动、闪红;死了出面板、重开正常。

---

## M1. 拾取 & 经验 & 升级

**目标**:打完怪能涨经验,升级可选一项。
**现状缺口**:敌人只受击杀但无掉落;玩家无经验值字段。

- 给 `Enemy` 加 `xp` 字段和 `onPostKill` 钩子(由 `Health` 组件在 `hp <= 0` 时触发)。
- 新增 `XpOrb` Actor:出生时随机小范围抖动散布,玩家走过(`onCollisionStart` + 玩家是 owner)被吸入加经验。
- 玩家持有 `xp / xpToNext`;升级时弹出"三选一"面板(在 M5 之前用最简占位列表 + 控制台打印),期间暂停逻辑。
- 升级选项先实现空壳:`{ 加血上限 / 移速 +x% / 攻击伤害 +x% }`,接口预留 M5 接入。

**验证**:杀 1~2 怪能看到经验小点,绕着走会吸过来;凑够升级停一帧弹三个文字选项,选完恢复。

---

## M2. 关卡结构:波次 / 倒计时 / 传送门

**目标**:有一局"短局 → 选奖励 → 下一关"的闭环。
**现状缺口**:敌人无限刷,无时间锚点。

- 新增 `LevelConfig`:每关有 `时长 / 敌人数密度曲线 / 精英时间点`。
- 把 `spawnTimer` 接入 `LevelState`,到点切到"传送门阶段":停止刷怪,场上敌人继续存在(玩家仍能刷经验),在地图中心生成 `Portal`。
- 玩家进传送门 → 推进 `currentLevel`,重置计时、敌人与经验点(玩家血量继承 / 升级继承)。
- 死亡回到第 1 关起点。

**验证**:开 1 局,到点出传送门,走进传送到下一关;关卡计时、场上密度有变化(可先只改"每关敌人数上限")。

---

## M3. 多样敌人

**目标**:同屏出现不同行为模式的敌人,玩家要根据局面选位。
**现状缺口**:只有匀速追踪的 `Enemy`。

- 把 `Enemy` 改成"数据驱动":每种敌人 = `EnemyKind { speed, hp, damage, behavior }`。
- 至少 3 种:
  - `Chaser`(现有)
  - `Dasher`:低血,定时冲向玩家,被近距离击杀也危险
  - `Shooter`:停住,周期性向玩家发射低速弹
- 抽 `behavior` 为策略对象(`onPreUpdate`),不写 if/else 大链。
- `spawnTimer` 按 `LevelConfig` 选种类与比例,密度随时间增加。

**验证**:第 1 关前期只看得到 Chaser;到 1 分钟开始混入 Dasher;第 2 关有 Shooter。

---

## M4. 多样武器

**目标**:升级选项里有真实"换武器"的诱惑。
**现状缺口**:玩家只有一把占位武器。

- 抽 `Weapon` 为可热替换组件:玩家身上始终持有一把,`swapWeapon(kind)` 切。
- 至少 3 种,共享 `Projectile` 与"攻速 / 弹道"配置:
  - `Pistol`(现有手感):直射、单发、低伤
  - `Shotgun`:扇形多发、近距离、攻速慢
  - `Aura`:不发射弹,在玩家身边生成伤害区间,无弹道
- 武器升级效果走"修饰器链",不复制实现:例如 `+damage` / `+pierce` / `+projectiles`。

**验证**:升级选项里出现武器时换上,弹道/手感真的变了;同一把武器多次升级能叠加。

---

## M5. HUD & 升级选择 UI

**目标**:玩家在画面上一眼看到血量、经验、计时、武器,升级有正经面板。
**现状缺口**:无任何 DOM/HUD 覆盖。

- 用 React 浮层(`/game` 路由里叠一层),从 Excalibur 通过 `EventEmitter` 暴露 `playerState`(hp / xp / level / weapon)与 `gameState`(timer / portalOpen)。
- 顶部条:血条 + 经验条 + 关卡与倒计时;底部:当前武器图标 + 攻速。
- 升级 / 商店面板:三张卡,每张 `{ name / description / icon }`,键盘 1/2/3 选。
- 暂停(ESC):全屏遮罩,引擎 `clock.stop()`。

**验证**:HUD 数值与引擎同步,升级三选一能键盘选;暂停时画面冻结、恢复时同步。

---

## M6. 角色 / 初始能力

**目标**:开局能选一个角色,各自有被动。
**现状缺口**:只有单一玩家。

- 在 `/` 主页加角色选择卡片,选完进 `/game`,把选择写进 `Engine.start` 前的 query/state。
- `Player` 通过"装饰器式 Buff"挂被动:`{ +maxHp, +speed, +might, 击杀回血, etc. }`,M0/M1 的升级效果也走同一套 Buff。
- 至少 1 个角色(数据先行,可选多个):例如"坦克(初始 +50% 血,移速 -10%)"。

**验证**:选不同角色进游戏,数值面板上看到差异,手感(攻速、移速)也明显不同。

---

## M7. 音效 & 视觉打磨

**目标**:从"能玩"到"想再开一局"。
**现状缺口**:贴图是占位 `player.jpg` / `enemy.jpg`,无音频。

- 换像素贴图 / 用 Excalibur 的 `SpriteSheet` 跑简单动画(玩家走、敌人受伤闪白、升级光环)。
- 命中 / 击杀 / 升级 / 传送四类短音效,Web Audio API 直接生成(或先用占位 WAV)。
- 击杀粒子:用 `Particle` 引擎,2~3 个预置就够了。
- HUD 微动效:受伤屏闪、升级卡片进场、传送门脉动。

**验证**:玩 5 分钟不掉帧(< 60 实体仍能稳 60fps),关键事件都有反馈。

---

## M8. 平衡 & 内容扩展

**目标**:一局 15~20 分钟、有重开动力。
**现状缺口**:无存档、无成就、无多关卡。

- 数值表:每关敌人数 / 血量 / 攻速 / 经验倍率集中到 `game/src/game/balance.ts`,便于单点调整。
- 关卡扩展到 5+ 关,每关一个主题(主题:怪物组合 / 精英 / 环境障碍)。
- 本地存档:最高关卡、已解锁角色;`localStorage` 即可。
- 调试开关:URL 加 `?debug=1` 显示实时数值面板(给开发自己用,不影响发布)。

**验证**:通 1 关后选角色 / 选升级的策略空间被实际用到;同一种打法不能每关都顺。

---

## 横切关注(每阶段都做)

- **代码组织**:逻辑、组件、资源、数据放 `game/src/game/` 下子目录(`actors/`、`components/`、`weapons/`、`enemies/`、`levels/`、`ui/`)。
- **类型安全**:所有公开配置(`WeaponKind`、`EnemyKind`、`LevelConfig`)走 `as const` + 联合类型;不允许字符串魔数。
- **测试**:纯函数(伤害公式、经验曲线、生成概率)走 `vp check` + 加单测;Excalibur 场景先靠手测清单,后续补 `vitest` + 引擎 mock。
- **性能**:敌人 / 投射物数量超过阈值时复用对象池(尤其投射物)。

## 暂不做的范围(避免范围蔓延)

- 联机 / 账号系统
- 关卡编辑器
- i18n(只做中文)
- 移动端触屏操作(先做键鼠,触屏留扩展点)
- 复杂剧情 / NPC 对话

---

## 建议执行顺序

1. **M0**(战斗)— 拿到第一份"能玩"手感。
2. **M1**(经验/升级)— 把"打怪→变强"循环接上。
3. **M2**(关卡)— 一局有头有尾。
4. **M5**(HUD/UI)— 此刻补 UI,体验立刻上一个台阶。
5. **M3 → M4**(敌人 / 武器)— 内容横向扩展,数据驱动为主。
6. **M6**(角色)— 用前面的 Buff 系统接入,改动小。
7. **M7 → M8**(打磨 / 平衡)— 收口。
